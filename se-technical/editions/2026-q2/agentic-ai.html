Who leads — Agentic-AI Protection

Radware leads agentic-AI protection with the market's first dedicated Agentic AI Protection GA product (Feb 2026) covering agent discovery, behavioral monitoring, memory-poisoning detection, and named M365 Copilot/Bedrock integrations; Cloudflare leads on MCP infrastructure governance with open-beta MCP Server Portals delivering tool-level access controls, shadow-MCP detection, and centralized policy enforcement; Akamai and Imperva are early-stage, offering adjacent API/LLM-layer controls with agent-specific MCP and runtime features squarely on mid-2026 roadmap.

  • No per-action policy syntax, write-blocking rules, or explicit destructive-operation controls are described in public product documentation
  • Tool-level access controls in MCP Server Portals enforce read-only vs. read/write tool permissions at policy definition time, providing a structural form of per-action governance

  • ML-based anomaly detection for unusual data exfiltration and suspicious tool usage is planned for coming months
  • No per-action policy enforcement for agent operations documented

  • No per-action enforcement or destructive-operation blocking is available in current GA products
MCP gateway / control plane
  • No MCP gateway or MCP control-plane capability is documented in Radware's GA Agentic AI Protection product

  • Administrators register internal and third-party MCP servers

  • Current Firewall for AI provides HTTP-layer prompt and response inspection without MCP-protocol awareness

  • Current Imperva AI Application Security operates as an LLM reverse proxy without MCP-layer awareness
Agent behavior monitoring & anomaly detection
  • Intent-Based Security is a named pillar of the GA Agentic AI Protection product, using patent-pending behavioral analysis to detect malicious or abnormal intent within agent interactions including multi-step and cross-agent behaviors
  • The Risk Graph Map continuously scores the agentic AI security posture and highlights multi-agent risk paths
  • Coverage aligns to OWASP Top 10 for Agentic AI and AIVSS scoring
  • Cloudflare AI Gateway provides real-time request logging, rate-limiting keyed to agent identity headers, and input scanning for injection patterns for all LLM-bound traffic
  • ML-based anomaly detection for unusual data exfiltration and suspicious tool usage across MCP Portal logs is described as coming months roadmap
  • Current GA monitoring is LLM-traffic-focused, not agent-behavioral-pattern analysis
  • Akamai API Security applies ML-powered API anomaly detection by learning expected request sequences and flagging deviations, which can be applied to MCP-style agent API traffic
  • Akamai also provides network-level agent monitoring via Guardicore Segmentation, limiting and observing agent communication paths
  • Neither capability is purpose-built for agent behavioral analysis
  • Imperva AI Application Security (beta) provides runtime monitoring of client-to-agent inbound traffic for OWASP Top 10 threats, distinguishing human users from AI agents in traffic analysis
  • The Thales-Google controlled availability tier adds 'real-time visibility' into AI traffic patterns
  • Full agent behavioral anomaly detection is a roadmap capability
Memory & RAG context-injection protection
  • Radware discovered and named the ZombieAgent attack class (Jan 2026): a zero-click indirect prompt injection vulnerability that implants malicious, persistent instructions into an AI agent's long-term memory or working context
  • The GA Agentic AI Protection product (Feb 2026) explicitly covers this threat via runtime behavioral analysis, making memory-poisoning detection a documented, named protection capability aligned to the ZombieAgent research
  • No dedicated memory or RAG context-injection protection capability
  • AI Gateway's DLP scanning and prompt injection detection (via AI Security for Apps WAF integration, roadmap) can intercept some injection payloads before they reach the model, but persistent memory poisoning or RAG data-source contamination protection is not a named feature
  • No memory or RAG context-injection protection documented
  • Akamai Firewall for AI focuses on prompt/response inspection at the HTTP boundary
  • The Thales AI Security Fabric roadmap reference to 'insecure RAG pipelines' is an Imperva item, not Akamai
  • Akamai has not published any product capability targeting agent memory or RAG context contamination
  • RAG Data Protection is a named component of the Thales AI Security Fabric roadmap: data protection in RAG pipelines using encryption and key management, sensitive data identification before ingestion, and security controls on LLM-to-data-source communication
  • The Thales SecurityBrief coverage confirms this is a 2026 roadmap item alongside MCP gateway
  • not yet GA
Agent identity & access scoping
  • Agent identity and access scoping are not explicitly documented as a separate capability in Radware's GA Agentic AI Protection product
  • The product addresses 'human-agent trust exploitation' as a threat vector (behavioral detection) and covers unauthorized data access via runtime monitoring, implying some identity-context awareness, but no IAM integration, credential binding, or least-privilege access-scoping interface is described publicly
  • Cloudflare Access integration enforces SSO, MFA, device posture, and contextual attributes (IP, location, device certificates) as preconditions for accessing MCP Server Portals
  • The Non-Human Identity (NHI) enhancements from Agents Week 2026 add resource-scoped API tokens, enhanced OAuth visibility, and managed OAuth for agents per RFC 9728 — providing least-privilege identity for agent-to-MCP connections
  • Cloudflare Mesh extends secure private network access to autonomous AI agents
  • Akamai provides identity-based access control for AI agents by combining access control with network segmentation: agents are assigned role-based identities, all interactions are proxied and monitored, and Guardicore Segmentation limits agents to predefined communication paths
  • These are existing Akamai security products applied to agent workloads, not a purpose-built agent-identity capability
  • End-to-end runtime access control covering 'unified access management across users, models, and data sources' is a mid-2026 roadmap item in the Thales AI Security Fabric
  • The Imperva for Google Cloud controlled-availability tier adds 'Verified Trust' for trusted AI agents and agentic browsers as a named capability, providing a first credential-validation step
Agent framework integrations (Bedrock Agents, M365 Copilot, OpenAI Assistants, LangGraph)
  • Radware's GA Agentic AI Protection product explicitly names Microsoft 365 Copilot, Microsoft 365 Copilot Studio, and AWS Bedrock as supported integration targets, with 'homegrown agents and more' noted

  • Cloudflare AI Gateway supports 70+ AI models across 12+ providers (OpenAI, Anthropic, Google, Groq, xAI, Alibaba Cloud, Bytedance, and more) via a single API endpoint, providing broad LLM-provider coverage for agent backends
  • MCP Server Portals work with any MCP-compliant server
  • No specific framework integrations for Bedrock Agents, M365 Copilot, OpenAI Assistants, or LangGraph are documented as named first-class integrations
  • No named agentic framework integrations documented
  • Akamai Firewall for AI works via edge, REST API, or reverse proxy deployment modes that are framework-agnostic, but no dedicated connectors for Bedrock Agents, M365 Copilot, OpenAI Assistants, or LangGraph are described in any public Akamai product materials
  • The Imperva for Google Cloud controlled-availability tier represents the closest named platform integration: protection for agents running in Google Cloud's agent ecosystem
  • No Bedrock Agents, M365 Copilot, OpenAI Assistants, or LangGraph integrations are named in current GA or near-term roadmap materials
